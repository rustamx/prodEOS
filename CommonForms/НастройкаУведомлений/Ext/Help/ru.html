Как запретить пользователям менять настройки по умолчанию

Здесь настраиваются уведомления, которые пользователи получат по умолчанию. Флажок Разрешить пользователям изменять настройки уведомлений определяет, разрешите ли вы пользователям менять состав этих уведомлений в персональных настройках.

  • Установлен – смогут подписаться или отписаться от любых уведомлений;
  • Не установлен – не смогут отписаться, изменить частоту и сроки уведомлений, которые настроил администратор.

Как настроить рассылку уведомлений

Перед выбором подписок на уведомления рекомендуем проверить настройки, без которых уведомления рассылаться не будут:

  1. Системная учетная запись почты должна быть настроена. Обязательно должен быть установлен флажок Использовать: Для "легкой" почты.
  2. Регламентные задания, отвечающие за рассылку уведомлений, должны быть включены:
    • Обработка произошедших бизнес-событий - формирует очередь уведомлений.
    • Уведомление пользователей о произошедших событиях - отвечает за непосредственную рассылку уведомлений.

После этого можно выбирать и настраивать подписки:

  1. Установите подписки на закладке Уведомления. При необходимости измените Частоту и срок отправки сообщений.
  2. Пропишите адреса электронной почты на закладке Адреса для уведомлений. Пользователи, у которых не указаны адреса электронной почты, не будут получать уведомления (в списке выделены красным цветом). Дополнительные адреса рассылки пользователи указывают в персональных настройках уведомлений самостоятельно.

Дополнительные настройки

  • Уведомления программы - настройка включает программные уведомления об ошибках и важных событиях, которые имеют прямое отношение к пользователю. Например, если пользователь настроил отложенный старт процесса, а при старте произошла ошибка - программа уведомит его об этом. 

  • Использовать выполнение задач по почте - настройка позволит выполнять задачи через обычный почтовый клиент, не заходя в "1С:Документооборот". Возможность может быть полезна при работе вне офиса. Для выполнения задач по почте должно быть включено регламентное задание Исполнение задач по почте.

Какие бывают подписки на уведомления

В каких случаях пользователь получит уведомление по подписке:

  • Ход выполнения процесса - если будут выполнены задачи процессов, автором которых является пользователь;
  • Остановка и прерывание процессов - процесс, автором или исполнителем которого является текущий пользователь, был прерван другим пользователем.
  • Поступление новой задачи - пользователю назначена новая задача;
  • Поступление перенаправленной задачи - пользователю перенаправлена задача;
  • Отмена выполнения задач – пользователь получит уведомление, если выполненная им задача отменена другим сотрудником;
  • Контроль просроченных задач - пользователь получит уведомление, если задача, исполнителем которой он является, просрочена;
  • Приближение сроков выполнения задач - подошел срок выполнения задач, где пользователь назначен исполнителем; Обратите внимание, что для работы данного уведомления должно быть включено регламентное задание Контроль приближения сроков задач.
  • Окончание срока действия документов - закончился срок действия документов, за которые пользователь назначен ответственным;
  • Приближение окончания срока действия документов - срок действия документов, за которые пользователь назначен ответственным, подходит к концу;
  • Появление ответа в форуме - новый ответ на сообщение форума или обсуждение предмета, с которыми связан пользователь (например, является автором сообщения или ответственным за обсуждаемый предмет);
  • Уведомление о просроченном контроле - срок контроля контрольных карточек пользователя истек;
  • Приближение контрольного срока - приближается контрольный срок заведенных полдьзователем контрольных карточек.
