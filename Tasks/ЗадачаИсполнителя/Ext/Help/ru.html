Задачи являются средством коммуникации между пользователями, а также используются для автоматизации процессов.   

Для создания новой задачи можно воспользоваться одним из процессов. Для создания задач на основании других объектов программы используйте подменю "Создать на основании". При этом объект-основание станет предметом задачи.

Если по задаче назначено несколько исполнителей, то для ответственного исполнителя карточка задачи содержит список состояний выполнения работ соисполнителей для контроля.

Для просмотра списка всех задач следует в разделе "Документы и файлы" или "Совместная работа" в панели навигации выполнить команду "Все задачи".

Для удобства просмотра можно установить отбор по автору, по исполнителю, просматривать только выполненные, невыполненные или все задачи, а также указать интервал дат по дате записи задачи.

С помощью команды Все задачи в карточке задачи можно посмотреть список всех других задач по данному процессу. Текущая задача в этом списке выделена голубым, выполненные задачи – серым, не принятые к исполнению – полужирным.

С помощью команды Принять к исполнению можно отметить задачу как принятую Вами. Принятая задача отображается обычным, а не жирным шрифтом. Если задача имела ролевую адресацию (т.е. была назначена не лично Вам, а роли, исполнителем которой Вы являетесь), то при выполнении команды Принять к исполнению адресация задачи будет  изменена и она будет адресована лично Вам, а не роли. Это удобно использовать для "захвата" задач в том случае, если исполнителями роли являются сразу несколько сотрудников. Отменить принятие к исполнению можно при помощи соответствующей команды в меню Все действия в карточке задачи.

Признак Задача остановлена отображается в списке задач или в карточке задачи в том случае, если был остановлен соответствующий процесс. Остановить процесс или сделать его активным может автор процесса или администратор. Задачи остановленных процессов не отображаются в списке Мои задачи и Задачи подчиненных.

Может быть Исключена из процесса.

Выполнение задач Согласование и Исполнение можно отменить. Для этого в панели действий списка Все задачи установите сортировку Показывать – Выполненные, откройте карточку задачи и нажмите Отменить выполнение. Если кнопка Отменить выполнение скрыта, значит процесс перешел на следующий шаг и отменить задачу уже нельзя. В этом случае, свяжитесь с автором процесса и договоритесь об остановке или перезапуске процесса.

Если в задаче не определен предмет, то исполнитель, при необходимости, может установить его сам с помощью команды "Установить предмет" в меню "Все действия" карточки задачи. При этом предмет будет установлен для всех незавершенных задач по текущему процессу, а также в самом процессе.

Задать вопрос автору задачи или другими сотрудникам можно напрямую из списка всех задач. Для этого предусмотрена команда Задать вопрос в меню Все действия. В открывшемся окне нужно ввести вопрос, установить его Важность, обозначить Срок ответа, прикрепить дополнительные файлы и посмотреть другие вопросы по данной задаче. Отозвать вопрос можно командой Остановить в карточке вопроса. Повторно отправить на рассмотрение – кнопкой Уточнить. Если команда для создания вопроса недоступна, скорее всего, вы забыли установить настройку программы Использовать рассмотрение вопросов выполнения задач.

Задачи можно помечать флажками. Флажки устанавливаются (и сбрасываются) при при помощи команды , расположенной на командной панели, или двойным щелчком мыши в столбце  списка задач. Флажок по умолчанию устанавливается двойным щелчком мыши в столбце . Флажок по умолчанию можно поменять в персональных настройках программы. Задачи, помеченные флажками, можно сортировать по столбцу . Флажки индивидуальны для каждого пользователя, т. е. пользователи видят только те флажки, которые установили сами.
