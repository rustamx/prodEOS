Загрузка электронной почты позволяет перенести почтовые сообщения из почтового ящика в справочник входящих, исходящих, внутренних документов или файлов.

Настройка загрузки:

  • Выберите почтовый профиль для загрузки сообщений.
  • Если профили еще не заведены, то их можно добавить в форме Настройка почтовых профилей.

    • При открытии формы будет использован выбранный профиль по умолчанию. При загрузке писем при помощи профиля Microsoft Outlook или MAPI в списке отображаются только непрочитанные письма.

    Для создания входящего, исходящего или внутреннего документа выберите почтовое сообщение и выполните соответствующую команду.

    Для сохранения вложения письма в справочник Файлы можно воспользоваться контекстной командой списка вложений Создать файл из вложения...

    Для удаления писем с сервера необходимо воспользоваться командой Удалить.

    В форме создания нового документа можно отметить файлы для импорта из письма.

    Если к письму присоединен файл с именем "1C-53898-2010.XML", то карточка входящего документа будет заполнена из содержащихся в нем данных. В этом файле содержится описание документа, соответствующее ГОСТу Р 53898-2010 "Системы электронного документооборота. Взаимодействие систем управления документами. Требования к электронному сообщению". Например, будут автоматически заполнены такие реквизиты карточки как "заголовок документа", "содержание", "регистрационный номер" и "дата регистрации" контрагента, "контактное лицо контрагента". При этом контактные лица контрагентов автоматически создаются при необходимости. Некоторые данные, поступившие от контрагента, добавляются в конец поля "Содержание" в карточке входящего документа, например, "гриф доступа" или "тип документа у контрагента".
