Том хранения файлов - это каталог на сетевом носителе, работающем по принципу клиент-сервер. Размещение в томах можно настроить согласно правилам.

Как создать том хранения файлов

Чтобы создать том хранения файлов:

  1. Введите название тома, заполнив поле Наименование
  2. Выберите группу из списка иди введите с клавиатуры вручную
  3. Введите полный путь для сервера 1С: Предприятия
  4. Выберите порядок заполнения, используя кнопки Вверх и Вниз
  5. Введите максимальный размер файлов на томе используя кнопки Вверх и Вниз
  6. При необходимости впишите комментарий
  7. Нажмите на кнопку Записать и закрыть, чтобы сохранить настройку

Примечание

Для использования томов, расположенных на сетевых дисках, сервер 1С:Предприятия 8 нужно запускать от лица доменного пользователя вида domain1\user1, имеющего права доступа к тому, указанному в виде UNC пути. Этот пользователь должен входить в группу Users локального компьютера, иметь права Run as service и Run as batch job и права на каталоги, в которых сервер 1С:Предприятия 8 хранит служебные файлы - журнал регистрации, каталог временных файлов и другие.

Ошибки доступа к томам помещаются в журнал регистрации и могут быть просмотрены администратором системы.

Как ввести полный путь для сервера под управлением Linux

Полный путь для сервера под управлением Linux можно ввести только с правами администратора.

Чтобы ввести полный путь, выполните следующие действия:

mkdir folder

Вместо folder напишите имя каталога. Например: в каталоге /home выполните mkdir shared. Создастся каталог shared с полным путем /home/shared.

mount -t cifs //server/share folder -o //domain/user, password=pwd

Где //server/share - UNC путь к другому компьютеру

folder - путь каталога на компьютере под Linux. Например: \home\shared, //domain/user - имя домена/пользователя (если нужно)

pwd - пароль пользователя (если нужно)

Если имя и пароль не нужны, команда выглядит так: mount -t cifs //server/share folder

После этого в пути к тому укажите folder. Например: /home/shared
