Как настроить автозапуск шаблона

Для этого нужно привязать шаблон к виду документов с помощью команды Назначен видам документов.

После выбора вида документа шаблон можно привязать к определенным организациям, назначить условие запуска, настроить интерактивный или автоматический старт.  

Интерактивный - программа запросит ваше согласие перед запуском процесса.

Автоматический - процесс запустится без вашего участия.

Как ограничить доступ к шаблону

Список пользователей и ролей, которые могут применять шаблон задается в Рабочей группе шаблона. Если рабочая группа не заполнена, то шаблон доступен всем.

При заполнении рабочий группы не забудьте внести в нее себя, иначе у вас не будет прав на использование данного шаблона.

Как настроить автоподстановку исполнителей

В качестве испонителей можно выбрать не только пользователей и роли, но и назначить автоподстановку. Для этого на закладке Настройки процесса нажмите Подобрать - Автоподстановка и выберите значения.

В этом случае, при старте процесса по шаблону список исполнителей заполнится значениями выбранной автоподстановки. Например, все руководители или подчиненные автора документа.

Как задать условие маршрутизации

Можно настроить процесс так, чтобы процесс передвигался по маршруту только при определенных условиях. Например, чтобы все договоры на сумму больше 100 000 руб. утверждал только директор. Для этого:

1. На закладке Настройки процесса нажмите кнопку Использовать условия.

2. Напротив каждого исполнителя появится гиперссылка Не установлено, по которой и настраивается условие.

Чем отличаются предметы процесса

Предметы процесса могут быть основными, вспомогательными и заполняемыми.

 Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания.

 Вспомогательный предмет содержит дополнительные данные, которые пригодятся при выполнении задач. Это может быть любой объект программы или файл с диска. По ходу выполнения задач процесса его состояние не меняется. 

 Заполняемый предмет требует введения каких-либо данных в процессе выполнения задачи. Задачу нельзя выполнить, пока предмет не будет заполнен.

Созданные предметы можно редактировать, менять их роли (основной, вспомогательный, заполняемый) и настраивать видимость для исполнителей.
