Как работать со списком исходящих документов

Исходящие документы служат для учета официальной корреспонденции, которую ваше предприятие отправляет внешним адресатам.

Чтобы быстро найти нужный документ в списке, используйте поля отбора:

  • Период,
  • Отправитель,
  • Вид документа
  • Организация.

Вы можете выбрать удобный режим просмотра:

  • Списком - отображается только список документов,
  • По категориям - рядом со списком в виде дерева отображаются категории. В список попадают документы только выбранных вами категорий.

Чтобы создать новый документ - нажмите Создать, чтобы отредактировать - Изменить.

В списках документов значок в колонке Задачи показывает наличие у документов сформированных задач.

Если настройка настройка программы Использовать категории включена, флажок  в колонке Категории отмечает документы, для которых установлены категории. Чтобы подобрать категорию к документу, воспользуйтесь кнопкой Подобрать категории в командной панели. При нажатии на нее открывается форма подбора категорий для выбранных в списке документов. 

Если настройка программы Использовать СВД включена, то в списке документов появляется колонка Документ поступил по СВД, в которой значком  помечены документы, пришедшие по СВД.

Если настройка программы Использовать электронные подписи включена, то в списке документов появляется колонка Подписан, в которой значком  помечены подписанные документы. При этом в карточке документа добавляются группа команд ЭП (содержит команду Подписать, с помощью которой выполняется удостоверение документа электронной подписью) и закладка ЭП, в которой редактируются и проверяются электронные подписи. Если документ подписан, то его редактирование невозможно. Для редактирования нужно сначала удалить все подписи. Текущий пользователь может удалять только свои подписи. Право удаления всех подписей принадлежит администратору. Подпись можно сохранить в файл (например, для пересылки по электронной почте), а также посмотреть сертификат подписи.

Документ можно отправить по электронной почте. Для этого в списке документов или карточке документа следует выполнить команду Отправить по электронной почте. На экран выводится форма отправки для ввода данных письма (получатель, тема, описание, вложенные файлы).

С помощью команд панели навигации можно открыть журнал передачи, просмотреть изменение состояния и историю переписки по документу.

При необходимости карточку документа можно распечатать.

В группе Печать (командной панели) и подменю Все действия доступны команды Штрихкоды. С их помощью вы можете:

  • Добавить внешний штрихкод - позволяет добавить произвольный штрихкод документу. В дальнейшем по этому штрихкоду можно найти документ.
  • Сохранить штрихкод - позволяет сохранить изображение штрихкода в виде графического файла *.JPEG на диск.
  • Штрихкод на наклейке - поможет напечатать наклейки с изображением штрихкода документа.
  • Штрихкод на странице - поможет напечатать штрихкода документа на странице.

Для работы штрихкодирования должна быть включена настройка программы Использовать штрихкоды.

Как работать с карточкой исходящего документа

Основные данные документа располагаются на вкладке Реквизиты. При создании нового документа следует задать:   

  • Вид документа - показывается, если настройка программы Использовать виды входящих документов установлена;
  • Организацию - в случае ведения документооборота для нескольких организаций (показывается, если установлена настройка программы Использовать учет по организациям);
  • Если документ отправляется в ответ на входящий документ, то в поле В ответ на указывается входящий документ;
  • Пользователя, подготовившего документ;
  • Руководителя (пользователя), подписавшего документ;
  • Получателя (контрагента или контактное лицо, для которого предназначен документ). Если требуется отправить документ нескольким получателям, то следует установить флажок Несколько получателей и указать получателей и адресатов в таблице, которая показывается на отдельной закладке;
  • Флажок Отправлен устанавливается, если документ уже отправлен, также устанавливается дата отправки;
  • Способ отправки документа;
  • Срок ответа (если требуется) - показывается, если настройка программы "Использовать срок ответа" установлена;
  • Гриф доступа - гриф определяет права доступа пользователей к входящему документу (если установлена настройка программы Использовать грифы доступа). Выберите подходящий гриф из числа заранее настроенных администратором.
  • Состояние документа - показывается, если настройка программы Использовать состояние документов установлена;
  • Вопрос деятельности - показывается, если настройка программы Использовать вопросы деятельности установлена;
  • В поле Краткое содержание (ниже поля Наименование) записывается текст, поясняющий цель отправки документа;
  • Предмет переписки - внутренний документ (если требуется).

В блоке Хранение следует заполнить значение поля Состав, описывающее количество экземпляров и листов документа. Состав заполняется через специализированную форму. Если документ помещен (списан) в дело, то указывается данные дела. При списании документа в дело подсчет общего числа листов выполняется по значениям полей Количество листов и Листов в приложениях в карточке документа. Поле учета состава документа показываются, если настройка программы Использовать состав документов установлена.

На отдельной закладке расположены списки связанных документов.

На закладке Визы находится список виз согласования данного документа. Вкладка доступна в карточке документа, если в настройках программы установлен флажок Использовать визы согласования. Визы согласования представляют собой записи о фактах и результатах согласования документов они могут создаваться вручную, либо автоматически из процесса Согласование. Создать новую визу или изменить существующую можно только, если документ находится в статусе На согласовании. При помощи команды Лист согласования можно распечатать результат согласования. При создании документа по шаблону список согласующих лиц шаблона копируется в список виз документа.

По умолчанию для всех пользователей установлена персональная настройка программы Использовать обзор документов. В этом случае, при открытии записанного документа отображается закладка Обзор, на которой расположена вся основная информация о документе, приложенные файлы, комментарий к документу и строка состояний.

Если настройка программы Использовать учет доступа к персональным данным включена, то в карточке входящего документа на закладке Дополнительно появляется список физических лиц, персональные данные которых хранит этот документ. Этот список физических лиц будет автоматически распространен на все файлы, присоединенные к данному документу и их версии. Также при этом будет осуществляться протоколирование доступа к полю Содержание входящего документа в журнал регистрации. Не рекомендуется использовать другие поля входящего документа (например, Наименование или Комментарий) для хранения персональных данных физических лиц.

Если документ был только что создан, на карточке документа появляется кнопка Заполнить по шаблону со значком . При нажатии на нее открывается форма выбора шаблонов документов данного типа, если для данного типа документов в системе есть шаблоны.

В зависимости от вида входящему документу автоматически присваивается регистрационный номер.

Для передачи исходящего документа другим исполнителям рекомендуется использовать процессы, доступные через меню Создать на основании, например - рассмотрение, исполнение или ознакомление.

Как контролировать исходящие документы

исходящий документ можно поставить на контроль. При этом будет создана контрольная карточка с записями о том, что контролировать, кого контролировать и на какой срок.

Есть несколько способов работы с контролируемыми документами:

  • Следить за всеми контрольными карточками в окне На контроле;
  • Контролировать сроки с помощью записей календаря;
  • Получать на электронную почту автоматические уведомления программы об истекающих и приближающихся контрольных сроках.

Контрольную карточку можно снять с контроля , вернуть на контроль , изменить истекший срок контроля , пометить на удаление.

Список ваших контрольных карточек не виден остальным сотрудникам (просмотр разрешен только вам, вашим руководителям и делегатам). Это позволяет использовать любые удобные для вас комментарии к контролируемым документам.

Если документ уже контролировался ранее, кнопка  откроет список всех контрольных карточек когда-либо созданных по документу. В списке можно посмотреть, кто сейчас контролирует документ и, при наличии прав, открыть контрольные карточки.

Как прикрепить к документу файлы

К каждому исходщему документу можно присоединить один или несколько файлов. Учет файлов возможен, если установлена настройка программы Использовать файлы у исходящих документов.

При перетаскивании файла мышью в карточку документа, к которому уже присоединен файл с таким же именем, программа либо предлагает сохранить версию имеющегося файла, либо не дает провести операцию, если имеющийся файл новее перетаскиваемого. Для присоединения к входящему документу второго файла с таким же именем воспользуйтесь одним из следующих способов:

  • переименуйте имеющийся файл и добавьте второй файл любым из возможных способов.
  • присоедините второй файл, используя кнопку Добавить.

    Как скопировать исходящий документ

    Документ можно скопировать. Для этого выберите документ-образец и выполните команду Все действия - Скопировать. При этом будет создан новый документ, основные поля которого заполнены данными из документа-образца. Внесите недостающие данные и исправьте (при необходимости) поля, заполненные из документа-образца.

    При копировании в создаваемый документ будут также скопированы файлы из документа-образца. Просмотреть и отредактировать файлы можно только после записи нового документа.

    Как ограничить доступ к документу

    Доступ к документу можно ограничить указаннием списка лиц на закладке Рабочая группа.

    Рабочая группа – это список пользователей и ролей, которые могут просматривать этот документ. Заполнить рабочую группу может любой пользователь, у которого есть права на просмотр и изменение документов. Если рабочая группа не заполнена, то документ могут просматривать все пользователи. При заполнении обязательно внесите текущего пользователя, иначе программа не даст записать документ.

    Если для вида документа указано Автоматически вести состав рабочей группы, то в рабочую группу автоматически добавляются авторы и исполнители процессов по документу, пользователи, указанные в самом документе.

    Для отключения автоматического добавления участников в рабочую группу конкретного документа можно указать Запретить автоматическое добавление участников рабочей группы на закладке Рабочая группа.
