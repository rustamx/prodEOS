План проекта содержит список проектных задач, которые объединены в иерархическую структуру и диаграмму Ганта проекта. Различные режимы просмотра плана проекта дают полное представление о проекте и его задачах на всех этапах выполнения. Проектные задачи вы можете редактировать как в самом списке, так и в карточке.

Как создать проектную задачу

Проектная задача – это действие или совокупность действий, которые направлены на достижение конечной цели проекта. Каждой задаче автоматически присваивается код СДР (структурной декомпозиции работ), который определяет ее уникальное место в проекте. Проектную задачу можно сделать вехой, то есть точкой, которая отмечает важное событие в проекте и используется для контроля хода выполнения. При создании проектной задачи вы должны установить ее плановую длительность, а также плановые даты начала и окончания. Затем исполнитель сможет отнести трудозатраты на проектную задачу или сам проект непосредственно из карточки своей задачи. Для этого  предусмотрены команды Включить хронометраж и Указать трудозатраты.

Все работы по созданию и организации исполнения проектной задачи вы можете выполнить в ее карточке. Чтобы начать создание проектной задачи, нажмите на кнопку Добавить командной панели.

Как организовать исполнение проектной задачи

Для организации исполнения проектных задач используйте процессы. Процессы по проектным задачам разграничены на Основные и Вспомогательные.

Основной процесс - это процесс, который направлен на выполнение проектной задачи. Трудозатраты на выполнение задач основного процесса попадают в фактические трудозатраты проектной задачи.

Вспомогательный процесс - это процесс, который не влияет на результат выполнения проектной задачи. Трудозатраты на выполнение вспомогательного процесса не попадают в фактические трудозатраты проектной задачи.

Чтобы создать Основной или Вспомогательный процесс по проектной задаче, используйте команды подменю Создать на основании командной панели.

Список всех процессов по задаче можно посмотреть в ее карточке при помощи команды Процессы панели навигации.

Как анализировать проект при помощи диаграммы Ганта

Диаграмма Ганта - это инструмент для иллюстрации расписания проекта. Диаграмма Ганта состоит из полос, ориентированных вдоль оси времени. Каждая полоса на диаграмме представляет отдельную задачу в составе проекта:

  • концы - моменты начала и завершения работы;
  • протяженность - длительность работы.

Вертикальной осью диаграммы является список задач. Выполненные задачи отображаются в виде отрезка. При помощи двойного щелчка мыши по полосе диаграммы вы можете открыть карточку задачи по проекту. Чтобы увеличить плановую длительность задачи, растяните отрезок задачи за края.

Отрезки диаграммы Ганта выделяются разными цветами:

  • черным - суммарные задачи (задачи, у которых есть подчиненные задачи);
  • синим - задачи, которые пока не выполняются, их дата начала пока не просрочена;
  • коричневым - задачи, которые пока не выполняются, однако дата их начала уже просрочена;
  • зеленым - задачи, которые уже выполняются, их дата окончания пока не просрочена;
  • красным - задачи, которые уже выполняются, однако дата их окончания уже просрочена;
  • серым - завершенные задачи.
