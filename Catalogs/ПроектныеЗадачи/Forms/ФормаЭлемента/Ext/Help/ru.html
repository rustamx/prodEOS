Как сделать проектную задачу вехой

Веха - это точка, которая отмечает важное событие в проекте и используется для контроля хода выполнения проекта. Веха имеет нулевую длительность и у нее не может быть исполнителей.

Чтобы сделать проектную задачу вехой, установите флажок Веха.

Как назначить исполнителей проектной задачи

Назначить исполнителей проектной задачи вы можете на закладке Исполнители. Если вы сформировали проектную команду, то выбор исполнителей проектной задачи будет возможен только из числа этих пользователей. При создании Основного процесса по проектной задаче список исполнителей процесса будет заполнен автоматически.

Как определить место задачи в плане проекта

Как организовать исполнение проектной задачи

Для организации исполнения проектных задач используйте процессы. Процессы по проектным задачам разграничены на Основные и Вспомогательные.

Основной процесс - это процесс, который направлен на выполнение проектной задачи. Трудозатраты на выполнение задач основного процесса попадают в фактические трудозатраты проектной задачи.

Вспомогательный процесс - это процесс, который не влияет на результат выполнения проектной задачи. Трудозатраты на выполнение вспомогательного процесса не попадают в фактические трудозатраты проектной задачи.

Чтобы создать Основной или Вспомогательный процесс по проектной задаче, используйте команды подменю Создать на основании командной панели.

Список всех процессов по задаче можно посмотреть в ее карточке при помощи команды Процессы панели навигации.

Общие сведения о проектных задачах

После сохранения в проектной задаче автоматически проставляется код СДР. Код СДР (код структурной декомпозиции работ) - это алфавитно-цифровой код, который определяет место задачи в структуре проекта.

Плановые даты проектных задач рассчитываются автоматически, исходя из длительности задачи, указанных предшественников и ограничений. Также вы можете указать задержку по отношению к предшественнику и тип ограничения в одноименном поле.
