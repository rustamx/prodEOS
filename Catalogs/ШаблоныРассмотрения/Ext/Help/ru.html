Общие сведения

В шаблоне процесса Рассмотрение можно указать различные данные, которые будут использованы при создании нового процесса на основании документа или при ручном заполнении нового-процесса из шаблона (команда Заполнить по шаблону меню Все действия в карточке процесса):

  • Наименование шаблона;
  • Наименование - наименование процесса, создаваемого по данному шаблону;
  • Описание;
  • Исполнитель, которому адресуется рассмотрение;
  • Срок;
  • Период отложенного старта; 
  • Важность.

Поля карточки процесса, заполненные из шаблона, становятся недоступными для ручного изменения пользователем.

Поля "Наименование шаблона", "Ответственный" и "Комментарий" носят информационно-справочный характер и не влияют на создание новых процессов по этому шаблону.

Любой пользователь, имеющий права на работу с процессами, может создать персональный шаблон процесса. В карточке персонального шаблона установлен флаг "Персональный шаблон". Персональный шаблон доступен для использования и изменения только его автором. Общие шаблоны процессов создаются ответственным за НСИ. Общие шаблоны доступны для использования всем пользователям, но для изменения или создания только ответственному за НСИ. Персональные шаблоны выделяются в списке шаблонов цветом текста.

Просмотр процессов, созданных по данному шаблону, выполняется с помощью команды "Процессы по шаблону" панели навигации карточки шаблона.

Чем отличаются предметы процесса

Предметы процесса могут быть основными, вспомогательными и заполняемыми.

 Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания. Состояние основного предмета изменяется в ходе выполнения задач согласования, утверждения, регистрации, рассмотрения, исполнения.

 Вспомогательный предмет, как правило, содержит дополнительные данные, помогающие при выполнении задач. Состояние вспомогательного предмета не меняется по ходу выполнения задач процесса. В роли вспомогательного предмета может служить любой доступный объект в программе или файл с диска.

Чтобы добавить предмет в шаблон, нажмите  Добавить на закладке Предметы.

Как настроить предмет

Созданные предметы можно редактировать, менять их роли и настраивать отображение для исполнителей.

Отредактировать свойства предмета можно кнопкой  .

Для изменения роли предмета служит кнопка . Это может потребоваться в том случае, когда при создании предмета его роль была выбрана ошибочно. Например, предмет по ошибке был выбран основным, и с его участием было настроено несколько условий перехода. После чего выяснилось, что для предмета больше подходит роль вспомогательного. Если предмет просто удалить, то все настойки будут потеряны. В этой ситуации удобнее изменить роль предмета.

Отображение предметов на этапах процесса настраивается в столбце  Видимость. Например, если не все вспомогательные предметы нужны для работы исполнителей, то эти предметы можно скрыть, сбросив флажок напротив нужного этапа.

Как задать условие участия исполнителя в процессе

Можно настроить процесс таким образом, чтобы исполнитель участвовал в нем только при определенных условиях.

Для этого служит кнопка Использовать условия на закладке Реквизиты. Если кнопка нажата, то напротив каждого исполнителя появляется гиперссылка Не установлено, вызывающая окно для установки условия.

Например, пусть все договоры на сумму, превышающую 100000, должен утверждать директор. Тогда в окне Установка условий в качестве предмета задается договор, а в качестве условия - "Сумма договора больше 100000 рублей".

Как настроить автоматическое применение шаблона

Для автоматического применения этого шаблона нужно привязать его к виду документов.

В этом случае при создании нового процесса на основании документа шаблон будет автоматически подставлен.

С помощью команды Назначен видам документов откройте список видов документов и дополните его записями с нужными видами. Для того, чтобы использовать условную привязку шаблона процесса к виду документов, необходимо указать, на основании какого условия маршрутизации будет осуществляться эта привязка. Если условие не указано, привязка будет осуществляться во всех случаях.

Как ограничить доступ к шаблону

Список пользователей и ролей, которые могут пользоваться шаблоном задается в Рабочей группе шаблона.

Если рабочая группа не заполнена, то шаблон доступен всем. Изменять шаблон может только пользователь, который указан в поле Ответственный.

При заполнении рабочий группы обязательно внесите в нее текущего пользователя, иначе программа не даст записать шаблон.
