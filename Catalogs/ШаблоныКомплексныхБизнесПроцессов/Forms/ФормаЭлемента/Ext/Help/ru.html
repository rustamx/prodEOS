Как добавить действия в шаблон комплексного процесса

Действия комплексного процесса - это набор типовых процессов, которые стартуют по настроенному порядку и при определенных условиях.

Чтобы добавить новое действие в шаблон комплексного процесса:

  1. Нажмите на кнопку Добавить действие, откроется помощник добавления действия.
  2. Двойным щелчком мыши выберите нужное действие (процесс) из списка.
  3. Заполните все необходимые поля в карточке действия (процесса).
  4. Нажмите на кнопку Записать и закрыть, чтобы добавить действие в шаблон комплексного процесса.

Чтобы еще быстрее добавлять действия в шаблон комплексного процесса, настройте шаблоны наиболее распространенных типовых процессов.

Если Вы добавляете в список действий шаблон типового процесса со всеми заполненными для старта полями, он будет сразу добавлен в список.

Как привязать шаблон комплексного процесса к виду документа

Можно привязать шаблон к виду документа. Таким образом, при старте комплексного процесса на основании документа, в помощнике создания процесса будет доступен список рекомендованных для этого вида документов шаблонов. Для этого:

  1. Выполните команду Назначен видам документов, в новом окне откроется список видов документов, к которым привязан этот шаблон.
  2. Нажмите на кнопку Создать, откроется карточка настройки шаблона процесса.
  3. Выберите вид документа из списка в поле Вид документа (клавиша F4).
  4. При необходимости установите условие в поле Условие.
  5. Нажмите на кнопку Записать и закрыть, чтобы сохранить настройку.

Можно запретить изменение процессов, созданных по этому шаблону. Для этого установите флажок Запретить изменение процессов, созданных по этому шаблону и для этого вида документов в карточке настройки шаблона процесса.

Как настроить автоматический старт комплексного процесса

Можно настроить автоматический старт комплексного процесса, который будет создаваться по этому шаблону. Например, чтобы сразу после создания заявления на отпуск автоматически стартовал комплексный процесс по обработке этого заявления.

  1. Нажмите на кнопку Назначен видам документов, откроется список видов документов, к которым привязан этот шаблон.
  2. Нажмите на кнопку Создать, откроется карточка настройки шаблона процесса.
  3. Укажите вид документа, которому будет назначен этот шаблон в поле Вид документа.
  4. Установите флажок Предлагать запуск при выполнении действия, чтобы активировать поле выбора действия:
    • Закрытие карточки только что созданного внутреннего документа – помощник автоматического запуска процесса появится после того, как Вы нажмете на кнопку Записать и закрыть в карточке документа.
    • Интерактивная регистрация внутреннего документа – помощник автоматического запуска процесса появится после того, как Вы зарегистрируете документ.
  5. Нажмите на кнопку Записать и закрыть, чтобы сохранить настройку.

Как настроить автоматический старт процесса по событию

Можно настроить автоматический старт комплексного процесса по событию. Например, чтобы комплексный процесс стартовал сразу после перерегистрации заявления на отпуск. При этом пользователь, который перерегистрировал документ, не увидит помощник автоматического запуска процесса и не сможет повлиять на процесс во время запуска.

Чтобы настроить автоматический старт процесса по событию, выполните следующие действия:

  1. Нажмите на кнопку Назначен видам документов, откроется список видов документов, к которым привязан этот шаблон.
  2. Нажмите на кнопку Создать, откроется карточка настройки шаблона процесса.
  3. Укажите вид документа, которому будет назначен этот шаблон в поле Вид документа.
  4. Снимите флажок Предлагать запуск при выполнении действия, если был установлен.
  5. Отметьте событие или события, после которых должен начаться процесс, в колонке События.
  6. При необходимости укажите условие, при котором будет осуществлен запуск процесса, в колонке Условие автозапуска процесса.
  7. Нажмите на кнопку Записать и закрыть, чтобы сохранить настройку.

Если для настройки недостаточно бизнес-событий, представленных в настройках шаблона процесса, воспользуйтесь настройкой автоматического старта процессов по событию с помощью правил автозапуска процессов (команда Все правила автозапуска процессов в подменю Все действия).

Как настроить права доступа к шаблону

Вы можете настроить права доступа к шаблону комплексного процесса посредством заполнения рабочей группы. Рабочая группа шаблона – это список пользователей и ролей, которые могут пользоваться этим шаблоном. Если рабочая группа не заполнена, то он доступен всем. При заполнении обязательно внесите текущего пользователя, иначе программа не даст записать шаблон. Изменять шаблон может только пользователь, который указан в поле Ответственный.

Чтобы заполнить рабочую группу шаблона комплексного процесса:

  1. Перейдите на закладку Рабочая группа шаблона.
  2. Нажмите на кнопку Добавить, станет активным поле выбора.
  3. Выберите пользователя или роль из списка или введите с клавиатуры вручную.

Чтобы еще быстрее заполнить рабочую группу, используйте кнопку Подобрать.

Чем отличаются предметы процесса

Предметы процесса могут быть основными, вспомогательными и заполняемыми.

 Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания. Состояние основного предмета изменяется в ходе выполнения задач согласования, утверждения, регистрации, рассмотрения, исполнения.

 Вспомогательный предмет, как правило, содержит дополнительные данные, помогающие при выполнении задач. Состояние вспомогательного предмета не меняется по ходу выполнения задач процесса. В роли вспомогательного предмета может служить любой доступный объект в программе или файл с диска.

 Заполняемый предмет предусматривается для одной из задач процесса и вводится на этапе ее выполнения. Если для задачи предусмотрен заполняемый предмет, то ее нельзя выполнить, пока предмет не заполнен. Для заполняемого предмета можно указать основание для заполнения, и задачу, в которой он должен быть введен. В шаблонах процессов также можно указать шаблон, по которому должен быть заполнен создаваемый предмет.

Чтобы добавить предмет в шаблон, нажмите  Добавить на закладке Предметы.

Как настроить предмет

Созданные предметы можно редактировать, менять их роли и настраивать отображение для исполнителей.

Отредактировать свойства предмета можно кнопкой  .

Для изменения роли предмета служит кнопка . Это может потребоваться в том случае, когда при создании предмета его роль была выбрана ошибочно. Например, предмет по ошибке был выбран основным, и с его участием было настроено несколько условий перехода. После чего выяснилось, что для предмета больше подходит роль вспомогательного. Если предмет просто удалить, то все настойки будут потеряны. В этой ситуации удобнее изменить роль предмета.

Отображение предметов на этапах процесса настраивается в столбце  Видимость. Например, если не все вспомогательные предметы нужны для работы исполнителей, то эти предметы можно скрыть, сбросив флажок напротив нужного этапа.

Как задать условие участия исполнителя в процессе

Можно настроить процесс таким образом, чтобы исполнитель участвовал в нем только при определенных условиях.

Для этого служит кнопка Использовать условия на закладке Реквизиты. Если кнопка нажата, то напротив каждого исполнителя появляется гиперссылка Не установлено, вызывающая окно для установки условия.

Например, пусть все договоры на сумму, превышающую 100000, должен утверждать директор. Тогда в окне Установка условий в качестве предмета задается договор, а в качестве условия - "Сумма договора больше 100000 рублей".
