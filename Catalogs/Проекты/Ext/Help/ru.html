Проект - это временное предприятие, предназначенное для создания уникальных продуктов, услуг или результатов. Используется для автоматизации исполнения плана проекта, а также для ведения аналитического учета всех данных программы (документов, файлов, мероприятий, писем, процессов и т.д.).

В результате в карточке проекта вы можете посмотреть:

  • список всех файлов, документов, мероприятий, которые относятся к проекту;
  • переписку по проекту;
  • трудозатраты на работу по проекту или его задачам и многое другое.

Для исполнения проектных задач используйте процессы. Процессы по проектам разграничены на Основные, направленные на выполнение проектных задач и Вспомогательные, которые не влияют на результат выполнения проектной задачи.

Текущее состояние проектных задач, соответствие фактических трудозатрат плановым вы можете узнать при помощи разных вариантов плана проекта.  Контролировать состояние проектных задач удобнее всего по диаграмме Ганта. Контролировать проектные задачи и анализировать трудозатраты вы можете при помощи отчетов Проекты и Проектные задачи.

Создавать проекты могут только пользователи, которые входят в группу доступа Руководители проектов.

Вы также можете загружать данные о проектах из Microsoft Project и организовывать их исполнение в программе. В качестве источника данных используйте файл проекта, выгруженный в XML.

 

Проекты можно поставить на контроль:

  • следить за сроком и ходом выполнения работ,
  • переносить контрольный срок,
  • контролировать отдельных пользователей,
  • снимать проект с контроля,
  • выводить аналитические отчеты,
  • смотреть, какие проекты контролируют ваши коллеги.

Программа сама отследит истекающие и приближающиеся контрольные сроки и сообщит вам об этом по почте.

Чтобы поставить проект на контроль, нажмите в командной панели карточки или списка проектов. При этом откроется Контрольная карточка проекта, в которой нужно установить срок, указать контролируемое действие и исполнителя. Если проект уже контролировался ранее, кнопка откроет список всех контрольных карточек когда-либо созданных по проекту. В списке можно посмотреть, кто сейчас контролирует проект, открыть контрольные карточки и, при наличии прав, изменить параметры контроля.

Состояние контролируемого проекта можно посмотреть в списке (колонка Контроль): - проект на контроле, контрольный срок не истек; - проект на контроле, контрольный срок истек; - проект снят с контроля. Значки с двумя означают, что проект находится на контроле у нескольких пользователей. Цвет по аналогии отражает состояние контроля у каждого из них.

После постановки на контроль рекомендуется настроить уведомления об истекающих или пропущенных сроках (в настройках программы) и следить за всеми контролируемыми документами в окне На контроле.
