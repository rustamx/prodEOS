Как управлять запуском процесса

Для процесса можно настроить:

  • Отложенный старт - запуск процесса через определенный промежуток времени. Например, сразу после заключения договора создать процесс "Утвердить макет здания" и отложить его запуск на месяц. Инструкция по настройке отложенного старта.  
  • Повторение - запускает процесс по указанному вами расписанию. Например, утверждение счета за канцтовары в последнюю пятницу каждого месяца. Инструкция по настройке повторения.

Как контролировать процесс

Процесс можно поставить на контроль. При этом будет создана контрольная карточка с записями о том, что контролировать, кого контролировать и на какой срок.

Есть несколько способов работы с контролируемыми процессами:

  • Следить за всеми контрольными карточками в окне На контроле;
  • Контролировать сроки с помощью записей календаря;
  • Получать на электронную почту уведомления об истекающих и приближающихся контрольных сроках.

Если настройка программы Использовать контроль объектов отключена, просто назначьте Контролера при создании процесса.

Разница между остановкой и прерыванием процесса

Остановленный процесс можно возобновить (команда Продолжить). Прерванный - нельзя.

При остановке процесса: на "паузу" ставится сам процесс и все его вложенные и подчиненные процессы (с задачами). Остановленные задачи не отображаются в списках задач.

При прерывании: прерывается сам процесс, все его подчиненные и вложенные процессы (с задачами), состояние предмета "Согласован" очищается. Действие необратимо.

Чем отличаются предметы процесса

Предметы процесса могут быть основными, вспомогательными и заполняемыми.

Основной предмет содержит важную информацию о задаче и выступает в качестве ее основания.

Вспомогательный предмет содержит дополнительные данные, которые необходимы для выполнения задачи. Это может быть любой объект программы или файл с диска. По ходу выполнения задач процесса его состояние не меняется.

Заполняемый предмет требует введения каких-либо данных в процессе выполнения задачи. Задачу нельзя выполнить, пока предмет не будет заполнен.

Что такое процесс утверждения

Процесс автоматизирует утверждение любых документов у ответственных сотрудников (ролей). Удобнее всего создавать процесс на основании одного или нескольких предметов. Для этого в любом списке или карточке объекта есть меню Создать на основании.

Результатов утверждения может быть два: утвержден и не утвержден. Неутвержденный документ можно отправить на повторное утверждение. Утвержденный - направить дальше по маршруту, например на регистрацию. Для этого в карточке ознакомления есть меню Создать на основании.

Документы можно утверждать по электронной почте, не заходя в программу (для этого должна быть включена персональная настройка Использовать выполнение задач по почте).
